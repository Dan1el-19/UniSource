# V2 Split Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Separate V1 legacy and V2 beta API contracts into independent URL families (`/` vs `/v2/*`), restore legacy routes from `main`, and update SDK V2 to use `/v2/*` paths exclusively.

**Architecture:** Copy V2 logic from current `routes/v1/*.ts` to new `routes/v2/*.ts` (plain V2 shapes, no legacy negotiation), restore 13 legacy routes from `main` into `routes/*.ts`, and rebuild `index.ts` with dual mount families. Update SDK V2 resources to use `/v2/*` paths, remove `X-Unisource-API-Version` header from transport.

**Tech Stack:** Hono, TypeScript, Zod, Cloudflare Workers, D1, pnpm workspace

**Source code references:**
- Source V2 logic: `apps/backend/src/routes/v1/*.ts` (13 files) — on current `feat/v2-split`
- Existing V2 routes to merge (not overwrite): `apps/backend/src/routes/v2/files.ts`, `apps/backend/src/routes/v2/folders.ts`
- Legacy routes to restore: `git show main:apps/backend/src/routes/<file>.ts` (13 files)
- Legacy index.ts reference: `git show main:apps/backend/src/index.ts`
- SDK V2 resources: `packages/unisource-sdk/src/v2/resources/*.ts` (13 files)
- SDK V2 transport: `packages/unisource-sdk/src/v2/transport.ts`

**Response shape transformation rules:**
| Current (routes/v1/) | Target (routes/v2/) |
|---|---|
| `itemOrLegacy(c, value, { file: value })` | `{ item: value }` |
| `itemOrLegacy(c, value, { folder: value })` | `{ item: value }` |
| `itemOrLegacy(c, value, { link: value })` | `{ item: value }` |
| `itemOrLegacy(c, value, { service: value })` | `{ item: value }` |
| `itemOrLegacy(c, value, { user: value })` | `{ item: value }` |
| `itemOrLegacy(c, value, { upload: value })` | `{ item: value }` |
| `itemOrLegacy(c, value, value)` (flat data) | `{ item: value }` |
| `listOrLegacy(c, values, {limit, next_cursor})` | `{ items: values, page: { limit, next_cursor } }` |
| `unpaginatedListOrLegacy(c, values, { items })` | `{ items: values, page: { limit: values.length, next_cursor: null } }` |
| `actionOrLegacy(c, {id, deleted:true,...}, {success:true,...})` | `{ item: { id, deleted: true, ... } }` |
| Bulk endpoints (files/folders) | Keep `{ processed, failed }` as-is |

## Phase 5: Rebuild apps/backend/src/index.ts

**Files:** Modify: `apps/backend/src/index.ts`

**Strategy:** Replace current monolithic `index.ts` (which imports from `routes/v1/*` and has global V2 wrapper) with a dual-family structure:
- Legacy mounts: import from `./routes/<file>`, use original `main` middleware structure
- V2 mount: import from `./routes/v2/index`
- V2 error wrapper: limited to `/v2/*`, not global
- V2 request ID guard: only for `/v2/*`
- No import from `./routes/v1/*`

Reference legacy structure: `git show main:apps/backend/src/index.ts`

- [ ] **Step 1: Write the new index.ts**

Key points:
1. Imports: legacy from `./routes/<name>`, V2 from `./routes/v2/index`
2. CORS: same as current (keep `X-Unisource-API-Version` in allowHeaders for now)
3. Global middleware: `loggerMiddleware`, `foreignKeysMiddleware` (same as now)
4. v2RequestIdGuard: change from `app.use('*', v2RequestIdGuard)` to `app.use('/v2/*', v2RequestIdGuard)`
5. Remove global V2 response wrapper middleware
6. Legacy mount points: exact same auth/middleware pattern as `main`'s index.ts
7. V2 mount: `app.route('/v2', v2Router)` with selective auth

8. V2 error transformation: add middleware on `/v2/*` that wraps non-V2 errors in V2 envelope (moved from global `app.use('*', ...)` to `app.use('/v2/*', ...)`)
